## FE 개발 시 주의할 점
- CSS는 head 태그에 넣는 것이 원칙!
- CDN(contents delivery network)으로 css 불러오는 것은 해당 CDN 서버가 언제 막힐지 모르기 때문에 서비스 단계에서는 위험한 행동이다.
- html에는 가급적 inline 으로 이벤트 만들지 않는 것이 좋은데 그래야 버그 발생 시 탐색 범위를 JS로 줄일 수 있기 때문이다. 즉 전체적인 모양은 html, 동작에 관한 것은 js로 완전히 이원화 하는 것이 바람직하다.
- css 선택자를 정할 때 class 명명법을 통일하는 것이 큰 프로젝트에서 버그 발생 위험을 줄인다.
- css 선택자 정할 때 너무 일반적인 용어로 만드는 것은 중복의 여지가 있어 바람직하지 않다.
- css에는 정답이 없지만 가능한 가장 간단한 방법으로 짜자!
- babel 같은 번역 장치를 쓰는게 아니라면 최대한 많은 브라우져에서 지원하는 js 문법을 사용하는 것이 좋다.
- 전역 부분을 더럽히지 말자! 보안이슈와 변수명이 겹칠 수 있는 위험성이 존재
- if문은 한줄이라도 {} 블럭으로 묶는게 가독성면에서 좋다!
- for loop의 종료 시점 값은 가능하면 변수로 한 번 저장하여 사용하는 것이 동작 예측면에서 좋다. 가능하면 forEach 문을 사용하는 것이 안정적이고 만약 특정 값에서 동작을 제외해야한다면, filter를 적절히 이용하여 제외할 값을 제외한 새로운 배열을 만드는 것이 부수효과를 없애기 좋다.
- 문자열 구성 시 이제는 template literal(백틱 사용해서 구성)이 대세!
- DOM에 변경을 가하는 코드는 가능하면 함수로 묶고 함수를 실행하는 형태로 만들자!(선언부 - 실행부의 분리!)
- DOM에 이벤트를 달았으면 해제하는 부분도 꼭 쓰자(beforeUnload)
- api요청 코드에는 짝을 맞춘 에러 처리 코드를 넣자
- ESLint와 같은 정적 분석기 강추!
